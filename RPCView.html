RPCView

2025.4.22

Overview

The RPCView application allows users:

  • Review, edit or create RPC descriptions
  • Register RPC with the applicationon context
  • Perform Testing of RPC
  • Review RPC implementation

    Application uses FileMan Delphi Components (FMDC) library to access VistA files. The information about files structure is provided by custom RPC.

    The application configuration file contains values of the user preferences and connection descriptors (IP,Port etc.)

    • Quick Start

    Starting the application

    		 Start application as any Windows executable - by double clicking the executable file name or using the shortcut.

    At the startup the application loads configuration data from the confirguration file.

    By default the configuration file is taken from the "AppData/FMView" directory. User may specify another location as the "config" parameter added to the shortcut target line: 

              /config=MyConfig.cfg

    If the configuration file contains access and verify codes values of codes are saved encrypted with the application "Master password".

    Enter "Master Password" to decrypt connection parameters:

    Connection to a server

    The "Select Server" dialog is opened automatically after master password is entered.

    To modify list of servers use "Edit", "Add", or "Delete" buttons.

    To connect an account - double click corresponding line of the list or click "Select" button.

    Upon successful connection the IP address and port of the server will be shown on the application main window status bar.

    Create an RPC group and add RPC to the group.

    To review and test an RPC select its name in the RPC List

    User Interface

    Screen Layout

    		 Main application window consists of:

  • Application Toolbar
  • RPC List
  • Information panel
  • Status bar

    • Application Toolbar provides quick access to main application functions. Buttons dublicate application main menu. Hints shortly describes button purpose.

    RPC List presents RPC names selected by user. RPC Names are saved in groups. List is saved in the application configuration file and reloaded on the application startup. Toolbar of the RPC List provides quick access to list editing functions.

    Note that list is not linked to any account - it stays the same regarles of server selected for connection.

    Inormation panel occupises most of the application window. Tabs located in the bottom area of the panel represent different aspects of the RPC:

    • Description
    • Tester
    • RPC Description editor
    • Implementation routine
    Description tab - contains RPC description from file #8994

    Tester tab allows perform simple testing - entering parameters and execution of RPCView

    RPC Description editor allows edit RPC record in file #8994.

    Implementation routine - access to the source code of the implementation routine.

    RPC Testing

    To test RPC - select "Test" tab:

    Parameters for the RPC execution can be entered in the input fields located at the top of the window. List of parameters will be created based on the RPC description in "remote Procedure"file (#8994). In case the description is incomplete - use buttons to add additional parameters.

    Buttons ... allow change order of the parameters.

    If needed - enter Application context in the corresponding input field.

    To execute RPC click "Exec"button

    Results of the execution will be shown in the area below the parameters.

    Note: copy of the execution results is saved in the application Log

    Preferences

    Installation

    Place the executable file `RPCView.exe` in a folder where you have write access.

    This folder should also contain the `Help` folder.

    Parameters

    Application support the following parameters:

    • /Config=your-config-file - name of the file containing configuration parameters. If not specified, application will try to use `RPCView.cfg`
    • /EDIT - enables editing of the RPC implementation routine (requires custom RPC)

    Custom RPC

    Application uses custom RPC for getting data from VistA. Sample code of the RPC (file `VWFMVIEW_1_42.m`) is included in the archive.
